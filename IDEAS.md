# Идеи для доработки

## 1. Нормальный README
Сейчас в README только заголовок(уже нет). Добавить описание проекта, скриншот вывода чека, инструкцию запуска (`php -S localhost:8000`).

## 2. Разделение данных и логики
`shop.php` совмещает данные и точку входа. Вынести `$products`, `$orders`, `$categories` в отдельный файл (`data.php` или `config.php`) — структура станет чище.

## 3. Интерактивная форма
Добавить HTML-форму, где пользователь выбирает товары и количество. Показывает знание `$_POST`, валидации входных данных и обработки запроса — превращает скрипт из «демо» в «приложение».

## 4. Персистентность заказов
Сохранять оформленные заказы в JSON-файл (`orders.json`) через `json_encode` / `json_decode`. Демонстрирует понимание хранилища без необходимости настраивать БД.

## 5. ООП-рефакторинг
Выделить классы `Product`, `Order`, `Cart` с методами вместо процедурных функций. Особенно актуально для Junior PHP позиции.

## 6. Unit-тесты
Написать тесты через PHPUnit для `formatMoney`, `formatDate`, `calculateTotals`, `sortOrdersByTotal`. Редкость в учебных проектах — сразу выделяет.

## 7. Корректный `wordwrap` для кириллицы
Текущий `wordwrap` считает байты, а не символы. В UTF-8 кириллица — 2 байта на символ, поэтому реальная ширина строки меньше ожидаемой. Реализовать через `mb_strlen` / `mb_substr`.

## 8. Фильтрация и поиск заказов
Добавить функции фильтрации по дате, по наличию конкретного товара, по диапазону суммы. Расширяет аналитический блок.

## 9. Экспорт чека
Возможность выгрузить чек в `.txt` или `.csv` файл через `file_put_contents`.
